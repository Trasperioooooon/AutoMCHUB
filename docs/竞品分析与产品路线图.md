# AutoMCHUB 竞品分析与产品路线图

> 调研日期：2026-07-12 · 目标：对标并超越 MCSL2

---

## 1. 对手画像

| | MCSL2 | MCSL Future（下一代） | MSL |
|---|---|---|---|
| 技术栈 | Python + PyQt5 + Nuitka | .NET 10 守护进程 + WPF/Tauri/Web 面板 | C# WPF |
| 热度 | 481 star，2026-06 仍活跃（v2.3.1.0） | 仅 122 star，架构重构起步期 | 千级 star |
| 王牌 | 多镜像下载（FastMirror/MCSL-Sync/极星/雨云）、1~2 秒 Java 扫描、三模式（简易/进阶/导入）、Python 插件系统 | 守护进程+远程管理架构 | **内置内网穿透**（OpenFrp/樱花/ChmlFrp/ME Frp 一键配置 + P2P 联机） |

## 2. 差距清单与借鉴点（按优先级）

### P0 —— 决定用户去留的四件事

**① 内网穿透集成（对标 MSL，当前最大缺口）**
- 现状：我们明确不做（v1 范围决策）；但"开完服朋友连不上"是新手第一痛点，MSL 靠这个立足。
- 借鉴：不自建穿透，接第三方 API——OpenFrp / 樱花 Sakura / ChmlFrp / ME Frp 都有开放接口；内嵌 frpc 托管（下载 frpc 二进制 → 生成 TOML 配置 → 进程托管），**完美复用我们现有的 dl 下载器 + proc 进程托管架构**，工作量约 2~3 天。
- ⚠️ 此项与最初"不做穿透"的范围决策冲突，需用户拍板。

**② 接入 FastMirror 聚合源，核心种类翻倍**
- MCSL2 靠 FastMirror 提供 Mohist / Arclight / Catserver / Banner（Forge+Bukkit 混合服，国内整合包玩家刚需）、Leaves / Folia / Spigot、代理端 Velocity / BungeeCord 等。
- 借鉴：FastMirror API v3 端点规律 `https://api.fastmirror.net/api/v3/{核心名}/{mc版本}/{构建}`（官方推荐用 API 取直链而非拼 URL）。在我们 `mcsrc` 里加一个 fastmirror.go 聚合源即可，每种新核心的启动方式几乎都是 `java -jar`，边际成本极低。
- 顺带获得：下载源三选一变四选一，可靠性再上台阶。

**③ 整合包一键导入（超车点，MCSL2 做得一般）**
- 模组服用户的真实工作流是"看到整合包 → 想开服"，手动装服务端 mod 是最劝退的环节。
- 借鉴：Modrinth `.mrpack` 是开放格式（modrinth.index.json 带服务端文件清单和下载直链，官方 API 免 key）；CurseForge manifest.json 需 API key 但可引导用户拖入已下载的整合包 zip。
- 落地：向导加"导入整合包"入口 → 解析清单 → 复用 dl 并发下载 mods → 自动识别所需加载器版本走现有创建流水线。

**④ 本机已装 Java 扫描（MCSL2 的招牌细节）**
- 借鉴其思路：扫注册表 `HKLM\SOFTWARE\JavaSoft`、`JAVA_HOME`、PATH、常见目录（Program Files\Java、.jdks、各大启动器的 runtime 目录），`java -version` 验证。
- 收益:已有合适 Java 时零下载，且是评测文章必比的项目。Go 实现约 150 行。

### P1 —— 体验拉开差距

| 功能 | 借鉴对象 | 备注 |
|---|---|---|
| 崩溃自动重启 + 定时重启/定时广播 | 面板类标配 | proc.go 已有状态机，加策略层即可 |
| 世界备份/还原（手动+定时 zip） | auto-mcs | 配合"删除实例前自动备份"防呆 |
| 玩家管理 GUI（在线/白名单/OP/封禁可视化） | MCSManager | 解析 whitelist.json/ops.json + 控制台命令双通道 |
| 控制台编码切换（GBK/UTF-8）+ 日志高亮过滤 + 命令历史 | MCSL2 论坛高频抱怨 | 我们已强制 UTF-8，老包乱码仍需兜底 |
| Modrinth 模组/插件浏览下载进实例 | MCSL2 插件市场 | Modrinth API 开放免 key |
| MOTD 彩色可视化编辑器（§ 色码） | MSL | 纯前端，半天工作量 |

### P2 —— 生态与长线

- **不要抄** MCSL2 的 Python 插件系统（其打包/兼容是长期包袱）。我们的架构红利：本地 HTTP API 本来就存在 → 直接开放 REST API + Webhook 文档，任何语言都能写扩展，天然是更好的"插件系统"。
- **远程管理近乎免费**：我们 UI 就是 Web——加一个"允许局域网访问 + 访问密码"开关，手机躺床上管服就实现了；这是 MCSL Future 重构两年想要的东西，我们改 20 行监听地址就有。
- 自动更新器（GitHub Releases + 国内加速代理）、多语言 i18n、亮色主题。

## 3. 已有优势（宣传时要反复敲打的点）

1. **单文件 8MB、零运行时依赖**：MCSL2 是 Python/Nuitka 打包上百 MB 且杀软误报频发；MCSL Future 要装 .NET 10 运行时。
2. **未来版本零维护**：Java 需求动态读 Mojang 元数据，26.x 新版本号体系已实测通过；竞品普遍靠硬编码表追版本。
3. **安装器网络加固**（--mirror + 预放置 + 重试）是实测国内网络后做的针对性工程，直接命中 Forge/NeoForge 安装失败这个全网高频求助话题。
4. Web 技术栈 UI：迭代快、天然支持远程化，且比 PyQt 好看。

## 4. 建议节奏

```
v1.1（1 周）：Java 本机扫描 + FastMirror 聚合源（Mohist/Arclight/Leaves/Folia/代理端）+ MOTD 编辑器
v1.2（2 周）：整合包导入（Modrinth 先行）+ 备份/定时任务/崩溃重启
v1.3（2 周）：内网穿透集成（OpenFrp + 樱花，若拍板做）+ 玩家管理 GUI
v2.0：开放 API/Webhook 文档 + 局域网远程访问 + 自动更新
```

竞争策略一句话：**不拼功能堆料，拼"最轻、最快、开箱即用、永不掉队（版本自适应）"**，用 P0 四项补齐火力后，以"8MB 单文件干翻百兆全家桶"作为传播记忆点。
